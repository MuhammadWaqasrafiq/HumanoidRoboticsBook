Appendix B: Troubleshooting Bible

title: "Appendix B: Troubleshooting Bible"

This appendix serves as a comprehensive troubleshooting reference for common issues encountered during the setup, development, simulation, and deployment of physical AI systems. It includes solutions related to ROS 2, Isaac Sim, hardware integration, Python environments, and system dependencies.

üê¢ Common ROS 2 Issues‚Äã

Problem: ros2 run command not found‚Äã

Cause: ROS 2 environment variables are not active in your shell.
Solution:

  1. Source the ROS 2 installation: 
    source /opt/ros/iron/setup.bash
  2. Add it permanently to ~/.bashrc: 
    echo "source /opt/ros/iron/setup.bash" >> ~/.bashrc
  3. Restart your terminal.

üß© Isaac Sim Errors‚Äã

Problem: Isaac Sim fails to launch or crashes frequently‚Äã

Possible causes:

  • Incorrect or outdated NVIDIA GPU drivers
  • Insufficient GPU RAM
  • Corrupted Isaac Sim installation

Solution:

  1. Verify NVIDIA driver: 
    nvidia-smi
  2. Ensure GPU meets minimum requirements (8 GB VRAM recommended).
  3. Reinstall or repair Isaac Sim from the Omniverse Launcher.
  4. Run from terminal to capture error logs.

ü§ñ Hardware Connection Problems‚Äã

Problem: Robot actuators or sensors are not responding‚Äã

Solution Checklist:

  • Verify wiring and polarity.
  • Ensure correct power delivery (voltage & amperage).
  • Check serial/Ethernet/USB ports.
  • Confirm correct baud rate, CAN ID, or IP address.
  • Test using a minimal standalone script to isolate the issue.

üêç Software & Python Environment Conflicts‚Äã

Problem: Libraries or dependencies conflict between environments‚Äã

Solution:

  • Create a dedicated virtual environment: 
    python3 -m venv .venv
    source .venv/bin/activate
  • Freeze dependencies: 
    pip freeze > requirements.txt
  • Use tools like Conda, Poetry, or pipx for clean isolation.

üîé Simulation Debugging Tips‚Äã

Problem: Inconsistent behavior between real robot and simulation‚Äã

Solution:

  • Validate physics parameters (mass, friction, inertia).
  • Check timestep & solver settings.
  • Ensure sensor noise models match reality.
  • Run unit tests on kinematics & control modules.

üöÄ Deployment & Runtime Issues‚Äã

Problem: ROS nodes crash after deployment‚Äã

Solution:

  • Check logs using: 
    journalctl -xe
    or 
    ros2 run <package> <node> --ros-args --log-level debug
  • Validate permissions: 
    chmod +x <node_file>
